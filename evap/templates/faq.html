    {% trans "These are the Frequently Asked Questions (FAQs)" %}

    {% trans "General" %}

    {% trans "What is evaluation?" %}

    {% blocktrans %} It's the assessment of all the courses at the HPI. It's the goal to improve the teaching quality by giving the lecturers feedback for their courses. {% endblocktrans %}

    {% trans "What is EvaP?" %}

    {% blocktrans %} It's this new platform - replacing the infamous EvaJ. {% endblocktrans %}

    {% trans "Where are the data coming from?" %}

    {% blocktrans %} The different information, imported into the system, are provided by the Studienreferat.
    After that the lectures, their delegates or the student representatives correct any mistakes.
    If you just found another error please contact the responsible lecturer or the student representatives. {% endblocktrans %}

    {% trans "Questionnaires" %}

    {% trans "How are questionnaires constructed?" %}

    {% blocktrans %} Every questionnaire of a lecture consists of different sub-questionnaires, e.g. about the contents or the teaching assistant.
    Each sub-questionnaire consists of one ore more questions. {% endblocktrans %}

    {% trans "What types of questions are there?" %}

    {% blocktrans %} There are two general types of questions:
    The first is called "Likert" and consists of five possible answers - from "strongly agree" to "strongly disagree". Additionally there is the option to give no answer.
    With the second type it is possible to leave text comments, if the other questions miss an interesting point. {% endblocktrans %}

    {% trans "How are questionnaires assigned to courses?" %}

    {% blocktrans %} In general the assignment is done by the student representatives. But the lecturers can add additional questionnaires themselves, if they have special questions. {% endblocktrans %}

    {% trans "Users" %}

    {% trans "How do I log into the system?" %}

    {% blocktrans %} If you are an HPI student or lecturer you can just use your usual credentials.
    If you are an external student or lecturer you can get an access token by Email. {% endblocktrans %}

    {% trans "How are users registered? Who is able to create new users?" %}

    {% blocktrans %} New users are created by the Student Representatives. Ask them if there is the need for new users. {% endblocktrans %}

    {% trans "What are delegates?" %}

    {% blocktrans %} As professors do not have much time, it is possible for them to elect one or more delegates, e.g. a secretary or an employee of the chair. These people then can edit the courses as the original lecturer could, i.e. add questionnaires and setup the correct assistants and evaluation period. {% endblocktrans %}

    {% trans "Publication of results" %}

    {% trans "How are the results publicised?" %}

    {% blocktrans %} The results are viewable in the EvaP system, after the exams for the course are done.
    Later the student representatives will hang out a hard copy of the results. {% endblocktrans %}

    {% trans "What information are in the printed export?" %}

    {% blocktrans %} These prints contain the exact same information as shown in this system.
    The involved lecturers can also view the comments, which were given to them. {% endblocktrans %}

    {% trans "How is the overall grade calculated?" %}

    {% blocktrans %} The overall grade is the arithmetic average of all sub-grades.
    All personal questions together are equally weighted to the questions about the course itself. {% endblocktrans %}

    {% trans "How many answers are necessary to assure the representativeness?" %}

    {% blocktrans %} Currently at least five (5) answers are necessary (except no answer) to assure the representativeness.
    If you do not agree with that write an Email or just come over to start a discussion with the student representatives. {% endblocktrans %}

    {% trans "Details for Lectures" %}

    {% trans "What data are necessary for a course and why?" %}

    {% blocktrans %} You can define all involved persons and their questionnaires. So it is possible for the students to give specific feedback.
    The evaluation period can be changed to fit to specific important dates of the lecture. {% endblocktrans %}

    {% trans "How should I configure the evaluation period of my course?" %}

    {% blocktrans %} The evaluation must not influence the assessment of the students. So there are different criteria for the period:

    • It should end before the written exam or the first oral exam.
    • It should be at least one week long.
    • It should start after one of the last lectures and not right at the beginning or after every lecture is held.
    Of course there are courses for which it is not possible to fulfil all of the criteria. Please contact the student representatives, if you are not sure. {% endblocktrans %}

    {% trans "What do the different course states mean?" %}

    {% blocktrans %} There are eigth (8) different states a course can be in:

    1. new: Right after import or creation by the student representatives all courses are in this state. The student representatives new pre check these information and changes the course into the next state. This is mainly "pending for lecturer approval". If this happens an Email is sent to the lecturers.
    2. prepared: In this state the lecturers can edit their courses, i.e. add and remove participants or change the evaluation period. If necessary the student representatives can approve the course, changing the state to "approved by FSR".
    3. lecturer approved by lecturer: After a lecturer approved a course the student representatives are checking the data and finally approve the course themselves.
    4. approved: After a course is approved (by the student representatives ultmatly) it is in this state. It will get into the next state automatically, if the start of the evaluation period is reched.
    5. in evaluation: In this state the participants can give their answers.
    6. evaluated: After the end of the evaluation period is reached, the course will get into this state automatically. Here the student representatives have to review all comment to remove inappropriate comments.
    7. reviewed: Only after the grades are published by the lecturers the courses can be published by the student representatives.
    8. published: Finally the results are here available in the system and the printout can be done, if nearly all courses are in this state.
    {% endblocktrans %}

    {% trans "When will I get which Email notice?" %}

    {% blocktrans %} At some point in the middle of the semester you get an Email with all your courses to edit them if needed.
    You will also get an Email after the results are available. {% endblocktrans %}

    {% trans "Which results do I see and what do they mean?" %}

    {% blocktrans %} You can see the average grade and standard deviation of all Likert questions.
    Additionally you can view the comments the students gave to you, the course or the other people involved in the course. {% endblocktrans %}

    {% trans "Details for Students" %}

    {% trans "How complete do I have to fill in the questionnaires?" %}

    {% blocktrans %} As complete as possible :)

    The results are only published if the representativeness is guaranteed. All Likert questions need to be answered (at least with no answer), but not one text comment question, to submit an questionnaire. {% endblocktrans %}

    {% trans "What happens to my comments?" %}

    {% blocktrans %} As we cannot assure the representativeness or anonymity here all comments are provided to all people involved in the course they were given.
    But the student representatives reserve the possibility to remove or change insults or other inappropriate texts. {% endblocktrans %}

    {% trans "How anonymous is my vote?" %}

    {% blocktrans %} The system guarantees that there is no connection between a vote and a user. It is just stored which user already submitted which questionnaire. Pay attention: We cannot assure anonymity regarding text comment questions as it is possible for the lecturer to guess from the contest or grammar of the text the student writing it.

    All text answers are given in an arbitrary order - so you should not refer to your own answers (like "see above" or "same as other teaching assistant"). {% endblocktrans %}
